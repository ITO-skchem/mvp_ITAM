담당자 - 서비스 매핑
1. 담당자 마스터 그리드에 서비스-담당자를 매핑할 수 있도록 "담당업무" 속성 컬럼을 추가한다.
2. 담당자 마스터의 역할은 "DT팀", "관리자", "Appl. 운영", "Infra 운영"으로 한다.
3. 서비스마스터에 추가할 속성 컬럼은 담당자의 역할만큼 만든다.
4. 담당자 마스터의 담당업무는 서비스마스터에 등록된 서비스명을 선택해서 입력할 수 있게 한다.
5. 담당자마스터에서 담당업무를 입력하여 저장하면 서비스-담당자 매핑이 갱신된다
6. 이제 더이상 서비스마스터에서는 담당자를 연결할 수 없고, 담당자 마스터에서 매핑한 값을 조회만 가능하다.
7. 담당자 마스터에서 선택한 역할에 따라 서비스 마스터의 DT팀, 관리자 등 속성에서 조회된다.

구성정보 - 서비스 매핑
1. 구성정보 마스터 그리드에 서비스-구성정보를 매핑할 수 있도록 "연결서비스" 속성 컬럼을 추가한다. 위치는 "구성명" 우측이다.
2. 구성정보 마스터 그리드의 연결서비스는 서비스 마스터에 등록된 서비스명을 선택해서 입력할 수 있게 한다.
3. 구성정보마스터에서 연결서비스를 입력하여 저장하면 서비스-구성정보 매핑이 갱신된다.

구성정보 - 컴포넌트 매핑
1. 구성정보 마스터 그리드에 구성정보 - 컴포넌트를 매핑할 수 있도록 속성 컬럼을 추가한다.
2. 추가할 속성 컬럼은 컴포넌트 마스터의 유형 만큼 만든다.
3. 추가할 속성 컬럼의 위치는 URL 우측이다.
4. 실제 매핑은 속성컬럼 필드를 선택하여 시작되고, 선택하면 구성정보 - 컴포넌트 매핑팝업이 뜬다.
5. 팝업 창은 어느 속성컬럼 필드를 선택하든 동일한 1개의 표준 팝업이다.
6. 표준 팝업은 아래와 같이 구성한다.
   1) 상단의 입력 텍스트 박스, 2) 중단의 컴포넌트명, 버전, 유형, 벤더명 선택 칸, 3) 저장/취소
7. 표준 팝업에서 동작은 아래와 같다.
   1) 텍스트 박스에서 컴포넌트 마스터의 컴포넌트 명 자동완성 일치 검증(오타 방지) 기능을 넣는다.
8. 팝업에서 저장버튼 누르면 팝업이 닫히면서 해당하는 속성컬럼에 값이 매핑/저장되고, 화면에서 값이 표시된다.
9. 팝업에서 취소버튼 누르면 팝업이 닫히면서 아무일도 일어나지 않는다.

시스템 통합정보 화면을 아래와 같이 수정한다.
이 화면의 역할은 각 마스터의 컬럼과 속성컬럼을 조합하여 개인화된 통합 View를 유연하게 만들어 보여주는 것인이다.
설계할때 보안, 성능을 충분히 고려하여야 한다.
화면 구성:
1. 좌측 조회항목 선택 영역
   - 서비스, 구성정보, 컴포넌트, 담당자 그룹을 트리 형태로 보여준다.
   - 각 그룹 아래에 기본 컬럼과 속성코드 항목을 체크박스로 보여준다.
   - 항목은 서버에서 제공하는 view_field_meta 데이터를 기준으로 렌더링한다.
   - 사용자는 여러 항목을 선택할 수 있다.

2. 우측 상단 조회조건 영역
   - 사용자가 체크한 항목 중 조건으로 사용할 항목을 추가할 수 있다.
   - 조건 행은 field_id, operator, value 구조로 관리한다.
   - data_type에 따라 operator 목록을 다르게 보여준다.
   - 문자열: contains, equals, startswith, endswith
   - 코드: equals, in
   - 숫자: equals, gte, lte, between
   - 날짜: equals, after, before, between
   - 버전: equals, gte, lte, lt
   - 여부: Y/N

3. 하단 결과 그리드 영역
   - Toast UI Grid를 사용한다.
   - 사용자가 선택한 체크박스 항목만 컬럼으로 표시한다.
   - 조회 결과는 AJAX API로 가져온다.
   - 페이징을 지원한다.
   - 엑셀 다운로드 버튼을 둔다.

구현 요구사항:
- integrated_view.html 템플릿을 만든다.
- 좌측은 체크박스 트리 UI로 구현한다.
- 우측 조건 영역은 동적으로 조건 행을 추가/삭제할 수 있게 한다.
- Toast UI Grid는 selected_fields 기준으로 컬럼을 동적 생성한다.
- 검색 버튼 클릭 시 AJAX로 /api/integrated-view/search/ 호출한다.
- 결과가 없으면 “조회 결과가 없습니다”를 표시한다.
- 오류 발생 시 Bootstrap alert로 메시지를 표시한다.
- Admin 사용자는 엑셀 다운로드 가능, 일반 사용자는 조회만 가능하게 한다.
