서비스-담당자 매핑
1. 담당자 마스터 그리드에 서비스-담당자를 매핑할 수 있도록 "담당업무" 속성 컬럼을 추가한다.
2. 담당자 마스터의 역할은 "DT팀", "관리자", "Appl. 운영", "Infra 운영"으로 한다.
3. 서비스마스터에 추가할 속성 컬럼은 담당자의 역할만큼 만든다.
4. 담당자 마스터의 담당업무는 서비스마스터에 등록된 서비스명을 선택해서 입력할 수 있게 한다.
5. 담당자마스터에서 담당업무를 입력하여 저장하면 서비스-담당자 매핑이 갱신된다
6. 이제 더이상 서비스마스터에서는 담당자를 연결할 수 없고, 담당자 마스터에서 매핑한 값을 조회만 가능하다.
7. 담당자 마스터에서 선택한 역할에 따라 서비스 마스터의 DT팀, 관리자 등 속성에서 조회된다.

코드 입력 자동 완성
담당업무 입력에서 서비스명 자동완성 일치 검증(오타 방지) 기능을 넣고, 엔터 또는 포커스가 이동하면 현재 선택된 값으로 자동완성시킨다.

(공통) 변경 라인별 저장
모든 그리드에 라인이 값이 수정될 경우, 추가될 경우 체크박스가 자동 체크된다.
저장 버튼을 누르면 체크된 라인 값만 저장된다.

(공통) 엑셀 업/다운로드 기능 추가
시스템 통합정보 - 엑셀 다운로드
서비스 마스터/구성정보 마스터/컴포넌트 마스터/담당자 마스터 - 엑셀 업로드 / 다운로드
엑셀 업로드 / 다운로드는 아이콘 처리하여 그리드 우측 하단에 우측 정렬로 배치한다. 아이콘 크기는 다른 text box 크기와 동일하게.
각 마스터에 코드 제약이 있는 경우, 엑셀 다운로드 시 1개의 sheet가 더 생성되어 사용하는 코드 그룹과 코드 리스트를 같이 내려준다. 

으로 재발 방지 팁(짧게):

템플릿 치환은 ApplyPatch/UTF-8 고정 편집만 사용
PowerShell Get-Content/Set-Content 대량 치환은 한글/특수문자 포함 파일엔 지양
아이콘 문자/SVG 변경 후엔 파일 diff에서 깨진 태그 여부를 먼저 확인

서비스 마스터의 분류 코드를 아래와 같이 다시 정리한다. 
경영지원, 기업문화, 영업, 재무, R&D, SHE, 생산, 품질, 구매, ERP, M365, G/W

담당자 마스터 테이블에 상주여부 컬럼을 추가한다. 
담당자 마스터 그리드에 역할 우측에 배치한다. 
코드그룹에 상주여부 코드그룹을 만들어 코드에서 선택하게 한다. 코드는 "상주" "비상주" 이다.

담당자 마스터의 "상태" 컬럼을 select로 바꿔라
(input-datalist)


마스터 화면 UI 규칙
1. 전체 CSS에서 일반 컬럼 길이 입력칸을 11rem 에서 8rem으로 변경한다.

2. 코드형 컬럼, input[list]. select 는 6rem 규칙으로 하고 추가 강제 규칙을 적용해서 확실히 고정해라

3. 엑셀 업로드/다운로드 버튼을 그리드 상단 툴바로 이동해서 하단 영역 자체를 제거하고 그리드 영역을 하단까지 최대한으로 넓힌다.

4. input[list]만 컬럼의 길이를11rem에서  6rem으로 줄여라

5. 엑셀 다운로드 양식 내 컬럼 순서를  화면의 그리드와 동일하게 맞춰라
속성 컬럼값과 같이 업로드해도 반영되지 않는 값은, 엑셀에 해당 열을 회색으로 표시해서 구분해라
코드에서 선택해서 입력하는 값은, 엑셀에 해당 열을 노란색으로 표시해서 구분해라

검색
1. 검색어 입력창에 검색어를 넣고 엔터 또는 검색을 눌러서 검색이 완료된 후, 
포커스가 검색어 입력창으로, 커서가 입력한 단어 끝부분에 가 있도록 해라
2. 검색 버튼 옆에 초기화 버튼을 만들어서 검색어 입력창을 모두 지우고 전체 조회한 후 검색어 입력창에 포커스가 가 있도록 해라
3. 검색을 눌렀을 때 테이블 자체 필드 외에 속성코드 값도 검색에 포함되게 해라. 
화면에 조회되는 값 기준으로 검색해야 하고, 검색성능을 고려해서 만들 것  
예를 들어 서비스 마스터 검색 창에 운영자 명 "민준홍"을 입력했을 때 MDM 시스템 행이 조회되어야 한다.
4. 검색 대상을 명확히 하기 위해, "서비스명 : PMO"와 같이 검색 컬럼을 지정하고 검색할 단어를 지정해서 검색할 수 있도록 해라
